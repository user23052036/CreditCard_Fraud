# Credit Card Fraud Detection

## Overview

This project implements a machine learning pipeline for detecting fraudulent credit card transactions using the well-known European cardholder dataset from September 2013. The dataset contains **284,807 transactions**, of which **492 are fraudulent**, making fraud detection a highly **imbalanced classification problem**.

The objective of this project is to build and evaluate multiple machine learning models to identify fraudulent transactions while addressing severe class imbalance. Several approaches are explored, including baseline models, oversampling techniques, and cost-sensitive learning.

The final workflow includes:

* Data exploration and preprocessing
* Feature scaling
* Baseline models (Logistic Regression and Random Forest)
* Handling class imbalance using **SMOTE**
* Advanced modeling with **XGBoost**
* Cost-sensitive learning
* Precision-Recall evaluation
* Threshold optimization for fraud detection

---

# Dataset Description

The dataset contains credit card transactions made by European cardholders in **September 2013**.

Key characteristics:

* **Total transactions:** 284,807
* **Fraud cases:** 492
* **Fraud percentage:** 0.172%

The dataset is extremely imbalanced, which makes traditional accuracy metrics misleading.

### Features

The dataset contains only **numerical features** because the original features were transformed using **Principal Component Analysis (PCA)** due to confidentiality reasons.

| Feature | Description                                                                      |
| ------- | -------------------------------------------------------------------------------- |
| Time    | Seconds elapsed between the transaction and the first transaction in the dataset |
| V1–V28  | PCA-transformed numerical features                                               |
| Amount  | Transaction amount                                                               |
| Class   | Target variable (0 = Legitimate, 1 = Fraud)                                      |

Because of the imbalance, the project primarily evaluates models using **Precision-Recall metrics** rather than raw accuracy.

# Workflow

## 1. Data Loading

The dataset is loaded using pandas and basic inspection is performed to understand the structure of the data.

Main steps include:

* Reading the CSV file
* Checking dataset dimensions
* Inspecting feature distributions

---

# 2. Train-Test Split

The dataset is divided into training and testing sets.

Important considerations:

* **Stratified splitting** is used to preserve the class distribution.
* **80% training data**
* **20% testing data**

This ensures that both datasets maintain the same fraud ratio.

---

# 3. Feature Scaling

Since PCA features are already normalized, only two features require scaling:

* **Time**
* **Amount**

Standardization is applied to these features using **StandardScaler**.

---

# 4. Baseline Models

Two baseline models are trained to establish reference performance.

## Logistic Regression

A linear classification model is used as the first baseline. Logistic regression helps evaluate how well the dataset performs with a simple model.

Key advantages:

* Fast training
* Easy interpretability
* Provides a baseline for comparison with more complex models.

---

## Random Forest

Random Forest is used as a non-linear ensemble model capable of capturing complex patterns.

Advantages:

* Handles nonlinear relationships
* Robust against noise
* Provides feature importance estimates.

---

# 5. Handling Class Imbalance

Fraud detection datasets are heavily imbalanced. Without addressing this issue, models tend to predict the majority class (non-fraud).

Two approaches are explored.

---

# 6. SMOTE Oversampling

SMOTE (Synthetic Minority Oversampling Technique) is applied to the training dataset.

SMOTE works by:

1. Identifying minority class samples
2. Generating synthetic examples
3. Balancing the dataset before model training

This helps models learn better fraud patterns.

---

# 7. XGBoost Model

XGBoost is used as the main gradient boosting model due to its strong performance on tabular data.

The model is trained under two scenarios:

### XGBoost with SMOTE

Training occurs on the oversampled dataset generated by SMOTE.

Advantages:

* Improves fraud recall
* Allows the model to learn minority patterns more effectively

---

### Cost-Sensitive XGBoost

Instead of modifying the dataset, this approach modifies the **training objective**.

A parameter called **scale_pos_weight** is used:

```
scale_pos_weight = number_of_non_fraud / number_of_fraud
```

This penalizes the model more heavily when it misclassifies fraudulent transactions.

Advantages:

* Handles imbalance without synthetic data
* Often more stable than oversampling.

---

# 8. Model Evaluation

Because fraud detection is highly imbalanced, traditional accuracy is misleading.

The project focuses on:

* Precision
* Recall
* F1 Score
* Confusion Matrix
* Precision-Recall Curve

The **Precision-Recall curve** is particularly important because it shows how well the model detects fraud while controlling false positives.

---

# 9. Threshold Optimization

Most classifiers use a default probability threshold of **0.5**.

However, in fraud detection, the goal is often to maximize fraud detection (recall) while maintaining acceptable precision.

The project explores different probability thresholds and analyzes:

* Fraud Recall
* Model Accuracy
* Precision trade-offs

Visualization helps identify the best threshold for operational deployment.

---

# 10. Visualization

Several visualizations are used to analyze model performance.

These include:

* Confusion matrices
* Precision-Recall curves
* Accuracy vs Fraud Recall plots

These graphs help interpret how different models behave when detecting fraudulent transactions.

---

# Key Insights

* The dataset is **extremely imbalanced**, making fraud detection challenging.
* Baseline models provide initial benchmarks but struggle with minority detection.
* **SMOTE significantly improves fraud recall**.
* **Cost-sensitive XGBoost provides a strong alternative** without altering the dataset distribution.
* **Precision-Recall evaluation is essential** for fraud detection tasks.
